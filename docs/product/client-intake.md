# Client Intake

## Required inputs

- service name
- staging or production URL
- browser/device targets
- login accounts
- top 3 to 10 critical flows
- current release goal
- known risky areas

## Nice-to-have inputs

- release note draft
- analytics or error logs
- previous bug list
- brand or accessibility requirements

## Intake questions

1. What user action matters most if it fails?
2. What workflow brings revenue or trust risk?
3. What changed in this release?
4. What must be true before you are comfortable shipping?
